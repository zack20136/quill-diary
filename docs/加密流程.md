# 加密流程

## 目標

寫入日記或附件時，產生可同時被 trusted device 與 `Recovery Key` 解開的 `LDJ2` 文件。

## 寫入流程

1. 產生隨機 `fileKey`
2. 用 `fileKey` 對正文做 `AES-256-GCM` 加密
3. 建立兩種 key slot
   - `device`
   - `recovery`
4. 組出 `LDJ2` header
5. 將 `magic + header length + header + ciphertext` 寫入檔案

## Key Slot

### Device Slot

- 由 Android Keystore 包裝 `fileKey`
- 供 trusted device session 使用
- slot 會標示目前使用的是 `plain` 或 `auth`

### Recovery Slot

- 由 Recovery wrapping key 包裝 `fileKey`
- `kdf` 參數會與 slot 一起寫入 header

## AAD

- `AES-GCM` 的 AAD 為整段 header bytes
- 只要 header 被竄改，正文驗證就必須失敗

## Manifest

- `manifest.json.enc` 是 recovery 驗證的首選目標
- 若 manifest 不存在，才 fallback 到其他可驗證的 `.enc` 檔
