# 產品與架構總覽

## 產品目標

QuillLockDiary 是一個離線優先的 Android 日記 App。

核心要求：

- 日記與附件必須以加密格式存放在本機
- 使用者可透過 trusted device 或 `Recovery Key` 重新取得存取權
- 索引資料庫只在可用 session 期間開啟
- 備份可輸出到本機或 Google Drive

## 架構原則

專案採用 `feature-first + 薄分層`：

- `features/`
  - 功能 UI 與功能狀態的正式入口。
- `domain/`
  - 穩定模型，例如 `DiaryEntry`、`UnlockedVaultSession`、`RecoveryMetadata`。
- `infrastructure/`
  - 外部邊界：加密、vault I/O、index database、Drive、裝置安全。
- `shared/`
  - 共用樣式、widget、provider 與工具。
- `app/`
  - Router、theme、app shell。

## 功能模組

### Session

- 管理啟動、解鎖、timeout、背景回前景後的 session 還原
- trusted state 失效時，直接收斂到 `recoveryRequired`

### Home

- 提供首頁、日曆、標籤管理、總覽
- 依賴解鎖後的 index 讀取列表與統計

### Editor

- 建立與編輯日記
- 管理附件暫存、標籤輸入、封面縮圖預覽

### Settings

- 顯示 session 狀態
- 建立 Recovery Key
- 開關裝置驗證
- 觸發本機與 Google Drive 備份流程

## 儲存結構

- `vault/recovery.json`
  - Recovery Key metadata 與 KDF 描述
- `vault/manifest.json.enc`
  - 加密 manifest，用於 Recovery Key 驗證
- `vault/entries/**/*.md.enc`
  - 加密日記
- `vault/assets/**/*.enc`
  - 加密附件
- `index/journal_index.sqlite`
  - 解鎖期間開啟的索引資料庫

## 破壞性策略

- 不保留舊版不相容 trusted device 格式
- 若 trusted state 無法被目前版本直接理解，會清除 trusted access 並要求使用 `Recovery Key` 重建
