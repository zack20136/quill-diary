# 模組參考

這份文件是 Quill Diary 目前程式結構的速查表。

它只回答兩件事：

- 這個型別、模組或 provider 是什麼
- 它主要負責什麼

流程、資料生命週期與使用者行為細節，請回到對應主題文件。

## 主要型別

| 型別 | 角色 |
|------|------|
| `UnlockedVaultSession` | 已解鎖日記庫 session，包含 `vaultId`、`trustedDevice`、可空的 `recoveryWrapKey`、可空的 `deviceSlotId`；`canUseRecovery` 代表目前是否可用 recovery wrapping key |
| `RecoveryMetadata` | `vault/recovery.json` 的明文 metadata，記錄 `vaultId`、建立時間、Recovery Key hint、版本與 KDF 參數 |
| `PendingAttachment` | 編輯器尚未正式寫入 vault 的附件；來源可來自記憶體 bytes、暫存檔路徑或 draft pending relative path |
| `EditorDraftSnapshot` | 編輯器當前欄位快照，用來判斷 dirty / empty 狀態；包含標題、日期、時間、標籤、正文、保留附件與 pending fingerprint |
| `EditorDraftPendingAttachment` | 草稿記錄內的 pending 附件摘要，保存 `relativePath`、`mimeType`、`originalFilename` |
| `EditorDraftRecord` | 寫入 `drafts/` 的草稿資料模型；除了欄位內容，還包含 `provisionalEntryId`、`createdAt`、`updatedAt` |
| `PersonalizationPreferences` | 個人化偏好聚合模型，包含語言、主題、自動鎖定、圖片品質與編輯器排版；提供 `materialThemeMode`、`materialLocale`、`sessionTimeout` |
| `AppLockStatus` | session 高層狀態：`uninitialized`、`unlocking`、`unlocked`、`locked`、`recoveryRequired`、`fatalError` |
| `SessionStartupPhase` | bootstrap / restore 後 lifecycle 週期：`bootstrapping`、`awaitingFirstBackground`、`resumeUnlockArmed` |
| `ForegroundResumeResult` | `SessionInactivityWatchdog` 在 `resumed` 時的回傳：`none`、`expired`、`waitingForSettle` |
| `SessionLockReason` | 進入 `locked` 的原因：`inactivity`、`manual`、`authFailed` |
| `PortableImportResult` | 可攜式匯入結果摘要，包含 `importedEntries`、`skippedFiles`、`skippedAttachments`，以及可選的 `failureCode` / `failureMessage` |
| `DriveConnectionState` | Google Drive 連線狀態摘要，含 `isConnected`、可選的 `email` 與 `displayName` |
| `DriveBackupFile` | Google Drive 上單一完整備份的摘要，含 `id`、`name`、`createdAt`、`sizeBytes` |
| `LocalBackupFile` | 本機 `backups/` 內單一完整備份的摘要，含檔名、路徑、建立時間與大小 |
| `BackupPersistResult` | 備份交付結果，區分 `success`、`inspectFailed`、`cancelled`，並附帶儲存路徑與訊息 |

## 核心模組

| 模組 | 職責 |
|------|------|
| `VaultRepository` | 正式日記庫初始化、trusted device session、Recovery Key 解鎖、entry / asset / manifest 讀寫、vault 修復、索引 attach 與 Keystore 同步 |
| `DeviceKeyManager` | Android Keystore 與 secure storage 互動；`hasTrustedKey` 需兩者都有效才算有 trusted device |
| `AppLockService` | 保存 `AppUnlockMode` 至 `app_lock_store.json`，並查詢 device credential / biometric 能力 |
| `AppSessionController` | 管理執行期 session 狀態、trusted unlock、manual lock、inactivity lock、restore 後 session 接手與 sensitive task 保護 |
| `SessionStartupCoordinator` | 冷啟動 bootstrap、trusted unlock 失敗分流、restore 後沿用既有 session 的決策 |
| `SessionResumeCoordinator` | 管理 startup phase、watchdog arm/disarm、lifecycle background 記錄、resume unlock 條件與 pending cleanup |
| `SessionInactivityWatchdog` | 管理背景逾時計時與 `resumed` 後約 500ms 前景穩定期 |
| `AppLifecycleSessionBridge` | 將 App lifecycle / pointer 事件接到 session；符合條件時約 350ms 後觸發 `unlock(source: lifecycleResume)` |
| `unlock_mode_policy.dart` | 解鎖模式能力檢查、trusted protection 一致性判斷、resume lock 訊息選擇 |
| `UnlockModeChangeService` | 安全切換解鎖模式，先同步 Keystore，成功後才落盤偏好 |
| `IndexDatabaseManager` | 依 session 開啟加密索引，衍生索引 key，並管理關閉、刪除與重建 |
| `EditorDraftStore` | 管理 `drafts/{draftKey}/` 下的加密草稿、pending 附件與預覽 materialize 檔 |
| `EditorFlowController` | 編輯器正式儲存、刪除、附件處理與草稿恢復的主流程協調器 |
| `SettingsFlowController` | 設定頁的 backup / restore / Drive / Recovery Key / repair / unlock mode change 流程協調器 |
| `TagStylesStore` | 管理 `vault/tag_styles.json` 的標籤樣式與 accent 顏色資料 |
| `PinnedEntriesStore` | 管理 `vault/pinned_entries.json`，保存首頁釘選狀態，並隨完整備份與還原保留 |
| `VaultArchiveIo` | 統一完整備份、還原、backup preview、portable import/export 的底層 I/O |
| `VaultTransferService` | 相容 façade，串接完整 backup / restore / 匯入匯出 / Drive 流程；新呼叫端以 capability service 為主 |
| `DriveBackupService` | Google Drive appDataFolder 備份上傳、列舉、下載、刪除、保留份數清理與帳號連線 |
| `UserPreferences` | 將 vault 外的 `app_preferences.json` 讀寫成個人化偏好 |
| `PickedFileMaterializer` | 將 `PlatformFile`、`content://` URI 或 bytes 材料化為可處理的本機檔案 |

## `domain/` 與共享值物件

| 模組 | 職責 |
|------|------|
| `VaultBackupPolicy`（`lib/domain/shared/`） | 管理 `backup_*.zip`、`markdown_*.zip`、`html_*.html` 的命名、辨識與保留份數（`retainCount = 5`） |
| `value_objects.dart` | 定義 `VaultId`、`EntryId`、`AssetId`、`DateOnly` 等跨模組共用值型別與產生器 |
| `DiaryEntry` | 正式日記聚合模型，供 vault、首頁、編輯器與匯出路徑共用 |
| `AssetAttachment` | 正式附件的領域模型 |

## `storage/` 子模組

| 模組 | 職責 |
|------|------|
| `VaultBackupIo` | 建立、檢查、預覽與還原 `backup_*.zip` |
| `PortableExportIo` | 產生 Markdown zip 與 HTML 匯出檔 |
| `PortableImportIo` | 匯入資料夾或 zip 內的 `.md`、`.html`、`.htm` 文件 |
| `EasyDiaryBackupImporter` | 辨識並匯入 Easy Diary 完整備份 zip |
| `OpenedZipArchive` / `archive_extract` | 處理 zip 開啟、選讀、解壓與一般檔案複製 |
| `ExternalDirectoryStore` | 記住最近一次使用的外部資料夾 |
| `ExternalDirectoryPicker` | 提供跨平台資料夾選擇；Android 寫入走 SAF |
| `deliverToExternalDirectory` | 將暫存輸出檔交付到外部資料夾 |
| `AndroidSafFileCopy` | 負責 Android SAF 串流寫入 |
| `PickedFileMaterializer` | 將使用者挑選的本機檔案或 URI 安全地 materialize 到 temp 檔案 |
| `VaultPathStrategy` | 提供 `vault/`、`index/`、`drafts/`、`backups/` 與各種正式路徑的單一計算入口 |

## Provider / Flow 速查

| Provider / 函式 | 職責 |
|------------------|------|
| `appSessionProvider` | 執行期 session 主狀態 |
| `sessionStartupProvider` | 啟動階段的一次性 session bootstrap |
| `effectiveAppSessionProvider` | 啟動狀態與執行期 session 的合併視圖 |
| `activeVaultSessionProvider` | 只在 `unlocked` 且 `session != null` 時暴露有效 session |
| `indexQueryableVaultSessionProvider` | 索引查詢使用的中介 session；在 `unlocking` 過程會暫留上一個已知 session |
| `finishRestoreSession()` | restore 寫入完成後重建 session；內部依情境走 `unlockWithRecovery()`、`resumeSessionAfterRestore()`、`recoveryRequired` 或 `bootstrapAfterRestore()`，並刷新依賴 |
| `beginPostRestoreStartup()` | restore 寫入完成後把 session 轉入 post-restore startup 狀態，並關閉舊資源 |
| `personalizationPreferencesProvider` | 個人化偏好主 provider |
| `storedAppLanguagePreferenceProvider` | 讀取使用者是否曾明確儲存 `zh` / `en` 語言偏好 |
| `watchPersonalizationPreferences()` | 在 widget 層讀取已載入的個人化偏好；未載入時回傳 defaults |
| `readSessionBackgroundTimeout()` | 依個人化偏好讀取目前背景逾時長度 |
| `unlockModeProvider` | 讀取目前 `AppUnlockMode` |
| `applyUnlockModeChange()` | 設定頁切換解鎖模式；內部會經過 `UnlockModeChangeService` 與 session 同步 |
| `editorDraftKeysProvider` | 提供目前仍有本地草稿的 entry key 集合 |
| `settingsFlowControllerProvider` | 設定頁 flow controller 入口 |
| `vaultRepositoryProvider` | 正式 vault repository 組裝點 |
| `vaultArchiveIoProvider` | 完整備份 / restore / portable I/O 組裝點 |
| `vaultBackupServiceProvider` | 本機備份、外部資料夾交付、Drive 上傳與保留數整理 |
| `vaultRestoreServiceProvider` | restore picker / precheck / Drive 下載 / restore 前準備 |
| `portableTransferServiceProvider` | Markdown / HTML 匯出與 portable import |
| `vaultTransferServiceProvider` | 相容 façade 組裝點；保留給過渡呼叫端，主要流程優先走 capability service provider |
| `editorDraftStoreProvider` | 草稿 store 組裝點 |
