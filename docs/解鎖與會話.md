# 解鎖與會話

啟動、背景逾時、本機受信任裝置還原，以及兩種解鎖模式與 `ResumeUnlockAction` 行為。

## 兩種解鎖模式

| 模式 | storage | 逾時回前景 | Keystore 槽 | 備援 |
|------|---------|------------|-------------|------|
| **裝置螢幕鎖**（預設） | `deviceLock` | 系統螢幕鎖對話框 | `deviceCredential` | 無 |
| **生物驗證** | `biometric` | 系統指紋／臉部 | `biometric` | 系統 prompt 內可改用裝置螢幕鎖 |

`deviceLock` 與 `biometric` 皆須裝置已設定螢幕鎖（`canUseDeviceCredential()`）。生物模式另須至少一種生物辨識已登錄。

舊版「無驗證 / plain trusted session」已移除；若偵測到 legacy slot，須以復原金鑰重新建立受信任裝置。

## Keystore 兩槽

Android `DeviceKeyManager` 對應兩種 alias（API 30+）：

| `KeystoreAuthKind` | 用途 |
|--------------------|------|
| `deviceCredential` | 系統螢幕鎖；`deviceLock` 模式 |
| `biometric` | 系統生物辨識；`biometric` 模式（Keystore 與 BiometricPrompt 均允許 `BIOMETRIC_STRONG \| DEVICE_CREDENTIAL`） |

生物模式不再維護獨立的 credential 備份 wrap；裝置螢幕鎖備援由 Android 系統驗證對話框提供。

## 架構邊界

| 元件 | 職責 |
|------|------|
| **`AppSessionController`** | session 狀態、`resumeAction`、呼叫 `VaultRepository.openTrustedSession` |
| **`SessionUnlockCoordinator`** | 監聽 `resumeAction`，自動觸發 `unlock()` |
| **`AppLockService`** | 持久化解鎖模式、查詢 `canUseDeviceCredential()` |
| **`DeviceKeyManager`** | Keystore wrap/unwrap |
| **`VaultRepository`** | 本機受信任 session、`ensureKeystoreMatchesUnlockMode` |

## `ResumeUnlockAction`

背景逾時或啟動解鎖失敗後，UI 與 coordinator 依此欄位決定下一步：

| 值 | 觸發條件 | 行為 |
|----|----------|------|
| `keystoreUnlock` | `deviceLock` / `biometric` 逾時回前景 | `unlock()` → 系統驗證對話框 |

使用者取消或驗證失敗時維持 `locked`，可手動按「重新驗證」再次觸發系統 prompt（生物模式 prompt 內仍可選螢幕鎖）。

## Session 逾時

- 門檻預設 5 分鐘（`SessionTimeoutPolicy`）。
- 逾時後清除 session 與解鎖資源，回前景時 `resumeAction = keystoreUnlock`，須重新完成系統驗證。

## 切換解鎖模式（設定）

程式入口：

- 讀取：`unlockModeProvider`（[`settings_providers.dart`](../lib/features/settings/providers/settings_providers.dart)）
- 寫入：`applyUnlockModeChange()`（[`unlock_mode_change.dart`](../lib/features/settings/unlock_mode_change.dart)）

流程：

1. 須已建立復原金鑰。
2. 選 `deviceLock` 或 `biometric` 前檢查 `canUseDeviceCredential()`。
3. 須先通過系統驗證並完成 `ensureKeystoreMatchesUnlockMode`（`targetMode`），**成功後**才寫入解鎖模式；在設定頁取消驗證則維持原模式且不鎖定 App（與背景逾時後在鎖定畫面取消驗證不同）。

## 與復原金鑰的關係

- 解鎖模式只影響**本機受信任裝置**的 Keystore 體驗，不取代復原金鑰。
- 本機受信任狀態失效、還原後日記庫不符、legacy plain slot → `recoveryRequired`，須輸入復原金鑰。
