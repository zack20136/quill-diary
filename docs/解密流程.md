# 解密流程

## trusted device 解鎖

1. 讀取 `recovery.json`
2. 確認目前裝置存在 trusted key
3. 讀取 wrapped recovery key 與 trusted device info
4. 用 device slot 解出 recovery wrapping key
5. 驗證 manifest 或其他加密檔
6. 建立 `UnlockedVaultSession`
7. 開啟 index database

## Recovery Key 解鎖

1. 使用者輸入 `Recovery Key`
2. 依 metadata 的 Argon2id 參數導出 recovery wrapping key
3. 用 recovery slot 驗證 manifest 或 fallback 檔案
4. 建立新 trusted device 狀態
5. 重建或同步索引

## trusted state 失效

以下情況都視為 trusted state 不可用：

- 舊版 slot id
- slot / device info 不一致
- wrapped recovery key 缺失
- keystore key 已失效
- 無法完成 unwrap

處理原則：

- 清除 trusted device access
- 將 app 狀態收斂到 `recoveryRequired`
- 不做舊版相容遷移

## 使用者取消驗證

如果目前啟用了裝置驗證，且使用者取消或驗證失敗：

- app 狀態停在 `locked`
- 不清除 trusted device access
