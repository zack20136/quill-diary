# 架構

Quill Diary 的靜態設計：產品目標、程式目錄、本機日記庫儲存與相容策略。

## 產品目標

- 日記與附件以加密格式存放在本機日記庫
- 使用者可透過本機受信任裝置或復原金鑰重新取得存取權
- 索引資料庫只在解鎖 session 期間開啟
- 首頁搜尋只要字串出現在標題、標籤或完整內文任一位置，就應找得到
- 備份可輸出到本機或 Google Drive

## 目錄分層

專案採用 `feature-first + 薄分層`：

| 目錄 | 用途 |
|------|------|
| `lib/features/` | 功能 UI、provider、state |
| `lib/domain/` | 穩定模型與 value object |
| `lib/infrastructure/` | 加密、儲存、索引、Drive、裝置安全 |
| `lib/shared/` | 共用樣式、widget、provider、工具 |
| `lib/app/` | Router、theme、app shell |

## `lib/infrastructure/storage/` 子結構

日記庫 I/O、備份與可攜式匯入匯出集中在 storage 層，並依來源拆分：

| 子目錄 | 職責 |
|--------|------|
| 根目錄 | `VaultRepository`、`VaultArchiveIo`（對外 facade）、`VaultTransferService`、路徑策略、還原前檢查 |
| `shared/` | 解 zip、MIME、檔案操作、`PortableImportResult` |
| `portable/` | 本 App `.jbackup`、Markdown/HTML 可攜式匯入匯出 |
| `import/<source>/` | 外部 App 備份匯入；目前為 `easy_diary/`，預留未來其他來源 |

設計原則：**外部來源各自子資料夾；本 App 可攜格式集中在 `portable/`**。模組速查見 [模組參考.md](./模組參考.md)。

## 功能模組

| 模組 | 職責 |
|------|------|
| Session | 啟動、解鎖、timeout、三種解鎖模式與 `ResumeUnlockAction`；本機受信任裝置失效時收斂到 `recoveryRequired` |
| Home | 首頁、日曆、標籤、總覽；依賴解鎖後的索引 |
| Editor | 建立與編輯日記、附件暫存、標籤、封面預覽 |
| Settings | session 狀態、建立復原金鑰、裝置驗證、標籤樣式持久化、本機與 Drive 備份 |

## 搜尋索引定位

- 首頁搜尋不直接掃描 vault 檔案，而是查詢解鎖期間開啟的索引資料庫
- 搜尋索引屬於衍生資料，不是權威來源；權威來源仍是 `vault/entries/**/*.md.enc`
- 新增、編輯、匯入日記時，都透過同一條儲存流程同步更新搜尋索引
- 索引重建時會重新掃描加密日記，重算摘要、標題搜尋字串、內文搜尋字串與標籤索引
- 搜尋採單一路徑子字串比對，不依賴 FTS tokenizer 或摘要 fallback

## 儲存結構

| 路徑 | 內容 |
|------|------|
| `vault/recovery.json` | 復原金鑰 metadata 與 KDF 描述 |
| `vault/manifest.json.enc` | 加密 manifest，復原金鑰驗證首選目標 |
| `vault/entries/**/*.md.enc` | 加密日記 |
| `vault/assets/**/*.enc` | 加密附件 |
| `vault/tag_styles.json` | 標籤目錄（名稱 + 可選 accent；備份含此檔；還原後與索引同步） |
| `index/journal_index.sqlite` | 解鎖期間開啟的索引（與日記庫分離，為衍生資料） |

## 破壞性策略

- 不保留舊版不相容本機受信任裝置格式
- 受信任狀態無法被目前版本理解時：清除本機受信任存取，要求以復原金鑰重建
- 偵測到舊 plaintext 索引時：直接刪除重建
- 搜尋索引 schema 不相容時：直接重建衍生索引，不保留舊索引內容
