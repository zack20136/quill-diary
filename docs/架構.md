# 架構

QuillLockDiary 的靜態設計：產品目標、程式目錄、本機儲存與相容策略。

## 產品目標

- 日記與附件以加密格式存放在本機
- 使用者可透過 trusted device 或 `Recovery Key` 重新取得存取權
- 索引資料庫只在解鎖 session 期間開啟
- 備份可輸出到本機或 Google Drive

## 目錄分層

專案採用 `feature-first + 薄分層`：

| 目錄 | 用途 |
|------|------|
| `lib/features/` | 功能 UI、provider、state |
| `lib/domain/` | 穩定模型與 value object |
| `lib/infrastructure/` | 加密、儲存、索引、Drive、裝置安全 |
| `lib/shared/` | 共用樣式、widget、provider、工具 |
| `lib/app/` | Router、theme、app shell |

## 功能模組

| 模組 | 職責 |
|------|------|
| Session | 啟動、解鎖、timeout、三種解鎖模式與 `ResumeUnlockAction`；trusted 失效時收斂到 `recoveryRequired` |
| Home | 首頁、日曆、標籤、總覽；依賴解鎖後的 index |
| Editor | 建立與編輯日記、附件暫存、標籤、封面預覽 |
| Settings | session 狀態、建立 Recovery Key、裝置驗證、本機與 Drive 備份 |

## 儲存結構

| 路徑 | 內容 |
|------|------|
| `vault/recovery.json` | Recovery Key metadata 與 KDF 描述 |
| `vault/manifest.json.enc` | 加密 manifest，Recovery Key 驗證首選目標 |
| `vault/entries/**/*.md.enc` | 加密日記 |
| `vault/assets/**/*.enc` | 加密附件 |
| `index/journal_index.sqlite` | 解鎖期間開啟的索引（與 vault 分離，為衍生資料） |

## 破壞性策略

- 不保留舊版不相容 trusted device 格式
- trusted state 無法被目前版本理解時：清除 trusted access，要求以 `Recovery Key` 重建
- 偵測到舊 plaintext index 時：直接刪除重建
