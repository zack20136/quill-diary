# 架構檢查

本文件記錄 2026-06-09 針對 `lib/` 與專案命名的低風險檢查結果。

## 結論

- 目前分層方向合理：`features/` 放 UI 與 provider，`domain/` 放穩定模型，`infrastructure/` 放加密、儲存、索引、Drive 與平台橋接。
- 本次已將 Dart 端命名常數集中到 `AppIdentifiers`，降低 MethodChannel、資料夾 namespace 與 package id 漏改風險。
- 本次沒有拆大型 UI 檔，避免在改名與註解任務中同時引入大量行為變更。

## 已處理

- 專案名稱統一為 Quill Diary；技術識別碼統一為 `quill_diary`、`quill-diary`、`zack20136.com.quill_diary`。
- Android Kotlin package 目錄已配合搬到 `android/app/src/main/kotlin/zack20136/com/quill_diary/`。
- 核心邊界已補註解：domain model、LDJ2 加密格式、device key、app lock、vault path、backup/export/import、Drive backup。
- `pubspec.yaml` 與 Android manifest 已移除與本專案無關的 Flutter 範本長註解。

## 後續風險

- `editor_page.dart`、`home_page.dart`、`home_page_widgets.dart` 仍偏大，且使用 `part` 拆檔；後續若要整理，建議獨立成 UI refactor 任務並搭配 widget tests。
- Android applicationId 已改為新 id，既有安裝、OAuth client、Keystore 與本機資料都會視為不同 app identity；上架與測試環境需同步更新。
- 本機資料資料夾從 `quill_diary` 開始使用；若未來要保留舊 app 資料，需要另做明確 migration。
