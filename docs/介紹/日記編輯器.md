# 日記編輯器

這份文件說明 Quill Diary 的日記編輯器實際提供哪些功能，以及編輯過程中的草稿、附件、搜尋索引與備份邊界。

它聚焦目前已實作的行為，不展開 Markdown 細節語法教學。

## 這個編輯器能做什麼

| 功能 | 說明 |
|------|------|
| 新建日記 | 可直接建立新日記，未儲存前先以本地加密草稿保存 |
| 編輯既有日記 | 既有日記預設先進入檢視模式，按編輯後切換到編輯模式 |
| 即時草稿保存 | 編輯模式下欄位變更後會自動寫入本地加密草稿 |
| 草稿還原 | 重新開啟同一篇日記或未完成新建時，若有草稿會先詢問是否還原 |
| Markdown 內容編輯 | 內文以 Markdown 編輯；含任務清單時切換為混合區塊編輯，並提供預覽模式 |
| 任務清單 | 可插入 `- [ ]` / `- [x]` 形式的任務項目，編輯模式下以互動區塊管理 |
| 預覽勾選 | 預覽模式中若內文含任務清單，可直接點選任務項目切換完成狀態 |
| 標題、日期、時間 | 可編輯標題、日期與建立時間；儲存正式日記時標題必填 |
| 標籤編輯 | 可輸入標籤，也可透過標籤編輯介面調整 |
| 圖片附件 | 可從裝置挑選多張圖片加入編輯器 |
| 一般檔案附件 | 可附加非圖片檔案 |
| 附件排序 | 圖片附件可在編輯器內重新排序 |
| 附件移除 | 可刪除既有附件或尚未正式寫入的待上傳附件 |
| 未儲存提示 | 有本地草稿時，日記列表與檢視模式會顯示「未儲存」標記 |
| 取消編輯 | 依是否有變更決定直接捨棄或跳出確認 |

## 編輯流程

| 時機 | 行為 |
|------|------|
| 開啟既有日記 | 預設進入檢視模式；按編輯才切到編輯模式 |
| 開啟新日記 | 直接進入編輯模式 |
| 編輯中 | 欄位變更後自動更新本地草稿 |
| 儲存 | 驗證標題後寫入 `vault/`，同步更新搜尋索引，並刪除草稿 |
| 取消 | 若相對已儲存內容無變更，直接清掉草稿；有變更則要求確認 |
| 再次開啟 | 若本地已有草稿，先詢問是否還原 |

新建日記的草稿 key 是 `__new__`。儲存成功後才會取得正式 `entryId`。既有日記則直接以 `entryId` 作為草稿 key。

## 主要編輯功能

### 檢視與編輯模式

- 既有日記先用檢視模式顯示內容
- 切到編輯模式後，才會開始追蹤欄位變更與草稿落盤
- 新建日記從一開始就是編輯模式
- 檢視模式與編輯模式會共用同一篇資料，但草稿只在編輯模式產生

### 內容編輯

- 內文以 Markdown 方式編輯；若內容含有任務清單（`- [ ]` / `- [x]`），編輯器會自動切換為混合區塊模式
- 混合模式下，每個任務項目可獨立編輯文字與勾選狀態，也可拖移調整順序
- 可透過底部工具列在游標位置插入新的任務項目
- 刪除最後一個任務項目後，會回到純文字編輯模式
- 編輯中含任務清單時，系統會在尾端保留一行空白文字行，方便繼續輸入；這是編輯輔助，不是內文的一部分
- 寫入草稿與正式儲存時，內文會先正規化：純文字會收斂首尾空白，任務清單則保留所需格式
- 提供預覽模式顯示排版後結果
- 支援標題、日期、時間、標籤與內文一起編輯
- 英文介面的日期欄位與時間戳記顯示使用 `yyyy/MM/dd`
- 儲存正式日記時一定要有標題
- 單純開新頁但完全空白的內容，不會保留成草稿

### 標籤

- 可直接輸入標籤 CSV
- 可開啟標籤編輯介面整理標籤
- 輸入時會自動去除空白、合併連續空格，並去除重複標籤
- 標籤順序會納入變更比對；調整順序也會視為有修改
- 標籤資料會納入草稿保存
- 正式儲存後，標籤會跟日記一起寫入日記庫並參與搜尋索引同步

### 圖片與附件

- 可從相簿或檔案挑選圖片
- 可附加一般檔案
- 新選取的圖片會依個人化頁「圖片品質」設定決定是否壓縮
- 新加入但尚未正式儲存的附件，會先放進草稿暫存區
- 圖片附件可在編輯器內重新排序
- 既有附件與待儲存附件都可以移除
- 圖片在檢視模式可開啟預覽圖庫
- 編輯模式下只要鍵盤開啟，就會暫時收起附件區與該列中繼資訊，讓主要編輯區保留更多空間

### 排版與預覽

- 編輯器與預覽會套用個人化頁的日記排版設定
- 可調整標題、內文字體大小、行距與段落間距
- 這些設定也會影響首頁列表與摘要的文字呈現
- 預覽模式支援一般 Markdown 排版
- 若內文含有任務清單，預覽會以任務項目呈現，且可直接點選任務項目切換完成狀態
- 預覽中的勾選變更會同步回寫內文

## 草稿如何運作

草稿是編輯器的一部分，不是獨立功能。它的用途是保留尚未正式寫入日記庫的編輯中內容。

### 儲存位置

草稿與日記庫、索引同屬 App 根目錄，但不放在 `vault/` 內。

| 路徑 | 內容 |
|------|------|
| `drafts/{draftKey}/draft.json.enc` | 加密草稿 JSON，包含標題、日期、時間、標籤、內文、附件資訊 |
| `drafts/{draftKey}/pending/` | 編輯期間暫存的待上傳附件 |

路徑策略定義在 [`vault_path_strategy.dart`](../../lib/infrastructure/storage/vault_path_strategy.dart)。

### 加密方式

- 草稿使用與日記庫相同的 **LDJ2** 加密格式
- 寫入與讀取都依賴目前已解鎖的 session
- 需要對應的 `vaultId` 與 `recoveryWrapKey`
- session 鎖定後，草稿也不能繼續讀寫

### 還原與清理

- 重新開啟編輯器時，若偵測到草稿，會先詢問是否還原
- 選擇不還原時，既有草稿會被刪除
- 正式儲存成功後，對應草稿會被刪除
- 草稿內不再引用的暫存附件會在寫入時順便清理
- 空白新建草稿會被自動捨棄，不留下無意義資料

## UI 訊號

- 日記列表與檢視模式會顯示紅色「未儲存」標記
- 這個標記代表該篇日記目前存在本地草稿
- 編輯模式本身不重複顯示這個標記
- 草稿存在狀態由 `editorDraftKeysProvider` 掃描 `drafts/` 後提供

## 與其他模組的邊界

| 項目 | 行為 |
|------|------|
| 搜尋索引 | 草稿不進搜尋索引；只有正式儲存日記後才同步 |
| 完整備份 | `backup_*.zip` 只封裝 `vault/`，不包含 `drafts/` |
| 還原 | 還原會覆寫 `vault/` 並重建索引，不主動清除既有草稿 |
| 解鎖狀態 | session 未解鎖時，草稿與正式日記都不能正常讀寫 |
