# 文件目錄

這裡是 Quill Diary 的核心文件入口。

如果你要理解這個專案，直接從這裡分流即可。核心閱讀先聚焦產品本體；Google Drive、Google Play、權限揭露與 Billing 另外集中在 `docs/介紹/google/`。

## 核心文件

| 文件 | 內容 |
|------|------|
| [架構.md](./架構.md) | 專案分層、主要模組、資料放在哪裡 |
| [模組參考.md](./模組參考.md) | 型別、Repository、Provider、storage 模組速查 |
| [解鎖與會話.md](./解鎖與會話.md) | 啟動、鎖定、三種解鎖模式、timeout、resume 行為 |
| [加密格式.md](./加密格式.md) | LDJ2、vault/檔案兩層金鑰、manifest 驗證與解密邏輯 |
| [索引資料庫.md](./索引資料庫.md) | 搜尋索引的開啟、同步、搜尋與重建 |
| [備份與還原.md](./備份與還原.md) | 完整備份、本機與外部流程、可攜式匯入匯出、還原後處理 |
| [日記編輯器.md](./日記編輯器.md) | 編輯模式、預覽、草稿、附件、儲存與取消編輯 |

## 怎麼看

- 想先理解整體結構：看 [架構.md](./架構.md)
- 想快速找程式責任邊界：看 [模組參考.md](./模組參考.md)
- 想理解資料怎麼被保護：看 [解鎖與會話.md](./解鎖與會話.md) 和 [加密格式.md](./加密格式.md)
- 想理解搜尋、備份、編輯器：直接進對應主題文件

## 補充

- `docs/` 內容以目前程式碼為準，不把規劃中的功能寫成已完成實作
- 若文件與程式碼不一致，以 `lib/` 內實作為準
- App 內法律文案：`legal_disclosures.dart`（含設定列標籤）、`privacy_copy.dart`（隱私摘要）；公開 URL 見 `lib/config/app_identifiers.dart`
- 完整法律頁面（GitHub Pages）：[`docs/privacy-policy.md`](../privacy-policy.md)、[`docs/third-party-notices.md`](../third-party-notices.md)
- Google 相關整備： [google/README.md](./google/README.md)
