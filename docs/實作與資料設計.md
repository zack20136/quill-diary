# 實作與資料設計

## 主要型別

### `UnlockedVaultSession`

- 代表目前已解鎖的 vault session
- 包含：
  - `vaultId`
  - `trustedDevice`
  - `recoveryWrapKey`
  - `deviceSlotId`

### `RecoveryMetadata`

- 對應 `recovery.json`
- 保存 vault id、建立時間、Recovery Key hint 與 Argon2id 參數

### `PendingAttachment`

- UI 已選取但尚未寫入 vault 的附件

## 主要模組

### `VaultRepository`

責任：

- vault 初始化
- trusted session 還原
- Recovery Key 設定與解鎖
- entry / asset 讀寫
- manifest 驗證
- index 同步與 rewrap 協調

### `DeviceKeyManager`

責任：

- 與 Android Keystore 溝通
- 管理 trusted device key
- 包裝 / 解開 recovery wrapping key

### `IndexDatabaseManager`

責任：

- 依 session 開啟 index database
- 關閉、刪除、重建 index 相關檔案

### `AppSessionController`

責任：

- 啟動流程狀態
- 解鎖、鎖定、timeout、背景恢復
- 將 trusted state 錯誤收斂為 `locked` 或 `recoveryRequired`

## Provider 設計

- `shared/providers/core_providers.dart`
  - 保存 app-wide 基礎依賴
- feature provider
  - 只管理 feature 自己的查詢、畫面狀態與互動

## 設計原則

- 跨 feature 的通用 UI 與工具才放到 `shared/`
- 只保留有明確流程價值的抽象層
- 複雜流程優先寫在 repository / coordinator，不建立只轉呼叫一次的 use case
