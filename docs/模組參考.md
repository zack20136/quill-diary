# 模組參考

主要型別、Repository 與 Provider 速查。

## 型別

### `UnlockedVaultSession`

代表目前已解鎖的日記庫 session，包含 `vaultId`、`trustedDevice`、`recoveryWrapKey`、`deviceSlotId`。

### `RecoveryMetadata`

對應 `vault/recovery.json`，保存日記庫 id、建立時間、復原金鑰 hint 與 Argon2id 參數。

### `PendingAttachment`

尚未寫入日記庫的附件：`sourcePath` 為編輯器選檔或匯入之本機圖；`bytes` 為匯入時解碼的 data URI。本 App HTML 匯入自 `<article class="entry">` 區塊解析附件。

### `EditorDraftSnapshot`

編輯器欄位快照，用於比對相對日記庫或上次落盤草稿是否有變更（`editorDraftIsDirty`）。定義於 [`editor_draft.dart`](../lib/features/editor/editor_draft.dart)。

### `EditorDraftRecord` / `EditorDraftPendingAttachment`

本地加密草稿的 JSON 模型：含標題、日期、標籤、內文、既有附件 id、pending 附件相對路徑與 provisional entry id。

### `ResumeUnlockAction`

目前僅 `keystoreUnlock`；見 [解鎖與會話.md](./解鎖與會話.md)。

### `PortableImportResult`

可攜式匯入（資料夾或 zip 解壓後）的統計結果，定義於 [`shared/portable_import_result.dart`](../lib/infrastructure/storage/shared/portable_import_result.dart)：

| 欄位 | 說明 |
|------|------|
| `importedEntries` | 成功寫入日記庫的篇數 |
| `skippedFiles` | 無法解析、空白略過或加密日記略過的次數 |
| `skippedAttachments` | 附件參考無法解碼或找不到本機檔案的次數 |
| `failureMessage` | 匯入失敗或無法完成時的說明（例如不支援的平台、Realm 不相容） |

設定頁 SnackBar 文案由 [`portable_import_result_messages.dart`](../lib/features/settings/portable_import_result_messages.dart) extension 格式化。

## 模組

### `VaultRepository`

| 職責 |
|------|
| 日記庫初始化 |
| 本機受信任 session 還原（含 legacy plain slot 偵測） |
| 復原金鑰設定與解鎖 |
| entry / asset 讀寫 |
| manifest 驗證 |
| 索引同步與 rewrap 協調 |
| `ensureKeystoreMatchesUnlockMode` |

### `DeviceKeyManager`

| 職責 |
|------|
| 與 Android Keystore 溝通（`deviceCredential` / `biometric`） |
| 管理本機受信任裝置 key |
| 包裝 / 解開 recovery wrapping key |

### `AppLockService`

| 職責 |
|------|
| 持久化 `AppUnlockMode`（`app_lock_store.json`） |
| `canUseDeviceCredential()`（螢幕鎖是否可用） |
| `keystoreAuthKindForCurrentMode()` |

### `IndexDatabaseManager`

| 職責 |
|------|
| 依 session 開啟索引資料庫 |
| 關閉、刪除、重建 index 相關檔案 |

### `EditorDraftStore`

| 職責 |
|------|
| 讀寫加密草稿 `drafts/{draftKey}/draft.json.enc` |
| `stagePendingFile`：複製選檔至 `pending/` |
| `delete`：刪除整個草稿目錄 |
| `listDraftKeys`：掃描仍有草稿檔的 key |

檔案：[`editor_draft_store.dart`](../lib/infrastructure/storage/editor_draft_store.dart)。詳見 [編輯器草稿.md](./編輯器草稿.md)。

### `TagStylesStore`

| 職責 |
|------|
| 讀寫 `vault/tag_styles.json` 中的標籤目錄（名稱與可選 accent 顏色） |
| 備份管線經 `VaultArchiveIo` 納入 `.jbackup` |
| 與索引資料庫 `tag_styles` 表同步 accent 快取；詳見 [索引資料庫.md](./索引資料庫.md) |

### `VaultArchiveIo`

對外 facade，委派至 `portable/` 子模組；設定頁與 `VaultTransferService` 仍只依賴此類，API 不變。檔案：[`vault_archive_io.dart`](../lib/infrastructure/storage/vault_archive_io.dart)。

| 職責 |
|------|
| 委派 `.jbackup` 備份／還原 |
| 委派 Markdown / HTML 可攜式匯出 |
| 委派資料夾或 zip 可攜式匯入 |

### `VaultBackupIo`

| 職責 |
|------|
| 建立 `.jbackup` zip |
| 備份健康檢查 |
| 還原、`verifyBackupRecoveryKey`、`peekBackupRecovery` |

檔案：[`portable/vault_backup_io.dart`](../lib/infrastructure/storage/portable/vault_backup_io.dart)

### `PortableExportIo`

| 職責 |
|------|
| Markdown 資料夾匯出 |
| Markdown zip 可攜式匯出（`writePortableExportZip`） |
| 選取篇數 HTML 匯出（`writeSelectedHtmlExport`） |

檔案：[`portable/portable_export_io.dart`](../lib/infrastructure/storage/portable/portable_export_io.dart)

### `PortableImportIo`

| 職責 |
|------|
| 遞迴匯入資料夾內 `.md`、`.html`、`.htm`（`importDocuments`） |
| zip 解壓後分流：先 Easy Diary，再 Markdown / HTML（`importDocumentsFromZip`） |
| 解析本 App front matter 與 HTML（[`html_import_parser.dart`](../lib/infrastructure/storage/portable/html_import_parser.dart)） |

檔案：[`portable/portable_import_io.dart`](../lib/infrastructure/storage/portable/portable_import_io.dart)

可攜式匯入支援三種來源：

| 來源 | 辨識 | 處理模組 |
|------|------|----------|
| 本 App Markdown | `.md` + front matter | `PortableImportIo.importDocuments` |
| 本 App HTML 匯出 | `<article class="… entry …">` | `PortableImportIo` + `html_import_parser.dart` |
| Easy Diary 完整備份 zip | `preference.json` + `Backup/Database/diary.realm_*` + `Photos/`（[`easy_diary_backup_layout.dart`](../lib/infrastructure/storage/import/easy_diary/easy_diary_backup_layout.dart)） | `EasyDiaryBackupImporter.tryImportFromExtractedRoot` |

zip 匯入流程（`importDocumentsFromZip`）：

1. 解壓到 temp
2. 先嘗試 Easy Diary 布局
3. 否則遞迴掃 `.md` / `.html` / `.htm`
4. 三者皆無可匯入內容 → 回傳 `zipImportNoEntriesMessage`

### `EasyDiaryBackupImporter`

| 職責 |
|------|
| 辨識 Easy Diary 完整備份布局 |
| 經 Android MethodChannel（`quill_diary/easy_diary_realm`）讀 Realm 快照 |
| 解析 `Photos/` 附件並寫入日記庫 |

檔案：[`import/easy_diary/`](../lib/infrastructure/storage/import/easy_diary/)（`easy_diary_backup_import.dart`、`easy_diary_photo_resolver.dart`、`easy_diary_realm_entry.dart`）

限制：完整備份 zip 匯入目前僅 Android；加密日記略過；Realm 讀取失敗時 `failureMessage` 說明原因。

### `VaultTransferService`

| 職責 |
|------|
| 設定頁 file picker：本機備份建立／還原 |
| Google Drive 連結、上傳、列出、下載 |
| Markdown / HTML 匯出、可攜式匯入（`importDocumentsWithPicker`） |
| 記住上次匯出／匯入目錄（`ExportSaveLocationStore`） |

檔案：[`vault_transfer_service.dart`](../lib/infrastructure/storage/vault_transfer_service.dart)

storage 子目錄總覽見 [架構.md](./架構.md#libinfrastructurestorage-子結構)。

### `AppSessionController`

| 職責 |
|------|
| 啟動流程狀態 |
| 解鎖、鎖定、timeout、背景恢復 |
| 將本機受信任狀態錯誤收斂為 `locked` 或 `recoveryRequired` |
| 設定 `resumeAction` |

### `SessionUnlockCoordinator`

| 職責 |
|------|
| 監聽 `resumeAction`，在 home / settings 自動觸發 `unlock()` |

## Provider

| Provider | 用途 |
|----------|------|
| `appSessionProvider` | 執行期 session 互動狀態 |
| `appStartupProvider` | 啟動一次性初始化 |
| `effectiveAppSessionProvider` | startup 與 local 狀態合併 |
| `activeVaultSessionProvider` | 僅在 `unlocked` 時暴露 session |
| `unlockModeProvider` | 設定頁讀取目前 `AppUnlockMode` |
| `applyUnlockModeChange()` | 設定頁切換解鎖模式（precheck → keystore 同步 → 寫入偏好）；見 [`unlock_mode_change.dart`](../lib/features/settings/unlock_mode_change.dart) |
| `shared/providers/core_providers.dart` | app-wide 基礎依賴（含 `appLockServiceProvider`、`editorDraftStoreProvider`） |
| `editorDraftKeysProvider` | 目前有本地草稿的 entry key；供首頁與檢視頁「未儲存」tag |

feature provider 只管理該 feature 的查詢、畫面狀態與互動。

## 設計原則

- 跨 feature 的通用 UI 與工具才放到 `shared/`
- 複雜流程寫在 repository / coordinator，不建立只轉呼叫一次的 use case
